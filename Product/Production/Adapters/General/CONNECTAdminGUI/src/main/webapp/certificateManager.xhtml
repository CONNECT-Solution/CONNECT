The Import Wizard allow users to create and import signed certificates.

User can start the process either by clicking on Creating a Self Signed Certificate or Generating a Certificate Signing Request (CSR). User also has the option of resuming the process using the Import SSL Certificate(s) option.

Note: The certificates created by the Import wizard are stored in the application properties directory under ~/importWizard/new/. User has to manually replace the existing certificates with the ones created by Import wizard and restart the server for changes to take effect.

This step creates a temporary certificate that will be used to replace the existing certificate in the KeyStore file.

Please skip this step and use Create Certificate Signing Request (CSR) step if an existing certificate will be used.
A Certificate Signing Request (CSR) will be generated. The CSR will be used to request a new SSL certificate from the Certificate Authority.
Click your Certificate Authority (CA) website links below. Follow their instructions to generate a signed Secure Sockets Layer (SSL) certificate. You will be asked to enter the authorization code, reference number and the Certificate Signing Request (CSR) generated in the previous step. When your certificate has been verified, you may resume using the Import Wizard to update Key and Trust Stores.
