This Import Wizard allows the user to add or import one certificate at a time.

The Import Wizard through a couple of steps makes a certificate, sets up the Certificate Signing Request (CSR) and allows for the certificate to be sent to be processed by a CA provider (Entrust or eHealth Exchange). Once the certificate has been verified by the CA provider, the Import Wizard can resume loading the Key and Trust Stores.

After the TrustStore complete button is pressed, the KeyStore and TrustStore files [stored in the importWizard/new folder] will need to be manually copied to replace the existing KeyStore and TrustStore files.

This step creates a temporary certificate that will be used to the replace the existing certificate in the KeyStore file.

You will be asked to enter the authorization code, reference number and the Certificate Signing Request (CSR) generated in the previous step. When your certificate has been verified, you may resume using the Import Wizard to update Key and Trust Stores.

To Replace the Server Certificate select the Alias and upload Server certificate; To overwrite the CA Root and CA Intermediate upload the CA Chain.
To override the existing CA Root and CA Intermediate certificates in the TrustStore: select the certificate and upload the CA chain.

Upon completion of the Import Wizard, the existing KeyStore and TrustStore files must be manually replaced with the ones provided in '/importWizard/new'. Once this is done, a server restart needs to be done to have the updates take effect.

If the Admin GUI deployment is hosted on a container server independent of CONNECT, the KeyStore and TrustStore must be manually replaced there as well.

The Import Wizard also stores a backup of the Key and Trust Store files (prior to the import) at /importWizard/backup folder.
